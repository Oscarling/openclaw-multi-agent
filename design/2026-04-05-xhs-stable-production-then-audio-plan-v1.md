# 2026-04-05 XHS 稳产优先 + 音频侧轨计划（v1）

## 目标

- 第一优先：保持当前图文发布主线稳定生产，不新增阻断点。
- 第二优先：在不影响主线交付的前提下，逐步引入“背景音乐 + 文字配音”能力。
- 增补要求：发布后到达复盘窗口时，Argus 自动触发复盘，不再等待人工下发指令。
- 原则：事件驱动，不用固定时间节点；任何增强都不能阻塞主线。

## 执行原则

- 主线（图文）与侧轨（音频）分离执行。
- 先保证“可持续发布 + 可追溯复盘”，再追求内容体验优化。
- 音频失败不影响当天主线发布。
- 关键闸门保持人工：最终发布按钮点击。
- 非关键环节尽量自动：发布回执登记、到窗采集同步、Argus 复盘触发。

## 角色分工

- `Argus`：主线生产与复盘（发布包、闸门、回执、复盘结论）。
- `Midas`：侧轨内容增强（口播稿压缩、BGM候选、配音文案润色）。
- `你`：最终发布确认与边界裁决（是否发布、是否进入增强版试点）。

## 阶段计划（事件触发）

### S1 主线稳产基线固化

- 触发条件：
  - A55 已完成（已满足）。
- 执行动作：
  - 统一执行链路：`xhs_argus_send(prep_strict) -> xhs_argus_prep_gate -> xhs_bridge_run -> 手工发布 -> xhs_post_publish_oneclick`
  - 每轮保留自动回执与复盘日志。
- 完成标准：
  - 连续 3 轮发布执行中，流程脚本无阻断性故障（允许内容表现波动，不允许流程失败）。

### S2 音频侧轨准备（不阻塞主线）

- 触发条件：
  - S1 完成。
- 执行动作：
  - 固化 `P_AUDIO_BRIEF` 模板：每篇图文自动生成 45~60 秒口播稿。
  - 固化 BGM 选择规则：仅使用平台可用或可商用来源，避免版权风险。
  - 产出“音频素材目录规范”与 evidence_ref 字段。
- 完成标准：
  - 任意一篇内容可在不改主线脚本的情况下产出“音频增强包”（不发布）。

### S3 音频增强试点（单篇）

- 触发条件：
  - S2 完成，且主线继续可用。
- 执行动作：
  - 选择 1 篇已发布主题，生成音频增强版（BGM + 配音）。
  - 单独发布或作为同主题扩展内容发布（不替换主线）。
  - 走同样回执与复盘口径。
- 完成标准：
  - 试点链路可闭环（准备 -> 发布 -> 回执 -> 24h 复盘），且无新增高风险告警。

### S4 是否合流到常态

- 触发条件：
  - S3 完成并输出复盘结论。
- 执行动作：
  - 比较“纯图文 vs 音频增强”在收藏、互动、关注上的相对变化。
  - 给出唯一结论：`继续侧轨` / `进入常态` / `暂停优化`。
- 完成标准：
  - 形成可执行结论并回填三本账。
