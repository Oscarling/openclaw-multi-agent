# M2-E5 阶段 C 放量策略卡（v1）

日期：2026-03-26  
阶段：M2-E5（阶段 C）  
目标：在受控前提下执行“小流量 -> 放量 -> 停机可回切”。

## 1) 流量梯度（事件触发）

| Phase | 批次上限 | 触发条件 | 失败触发 |
|---|---:|---|---|
| C1（小流量） | 5 | Stage-C 预检通过，且人工闸门确认 | 单批失败数 >= 1 立即停机 |
| C2（受控放量） | 20 | C1 连续 2 批成功，且成功率 >= 95% | 单批失败数 >= 2 或成功率 < 90% |
| C3（扩大放量） | 50 | C2 连续 2 批成功，且成功率 >= 97% | 单批失败数 >= 3 或成功率 < 92% |

## 2) 停机与回滚阈值

- 停机触发：
  - 任一批触发失败阈值。
  - 回执缺失或回执字段不完整。
  - 发现越权发布或非白名单账号发布。
- 回滚动作：
  - 立即切换回人工发布链路。
  - 记录 `batch_id/release_id/account_id/operator/ticket_id`。
  - 回填 `DECISIONS.md` 与阶段 C DoD。

## 3) 人工闸门要求

- 每批执行前必须有人确认（operator + ticket）。
- 每批必须回填发布回执，禁止无回执成功判定。
- 仅白名单账号可进入阶段 C 链路。

## 4) 通过判定（阶段 C 准备态）

- 放量梯度定义完成，且阈值可执行。
- 停机与回滚动作明确可操作。
- 与阶段 B 回执口径一致，可追溯。
