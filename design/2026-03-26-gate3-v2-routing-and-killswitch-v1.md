# 2026-03-26 Gate-3 v2 命中策略与 Kill-Switch 说明（v1）

更新时间：2026-03-26  
范围：Gate-3 prompts.chat 角色优化的试运行入口与熔断策略  
状态：正式评审通过后可执行（当前不改默认入口）

## 1) 设计目标

- 保持默认外部入口不变：`steward` 仍为唯一对外入口。
- v2 能力默认不自动命中，避免真实入站流量被新能力误伤。
- 出现越权/质量回退时可在分钟级回切到稳定口径。

## 2) v2 命中策略（默认关闭）

- 策略 A（默认）：仅允许显式 `--agent *_v2` 调试命中。
- 策略 B（受控）：仅在测试绑定通道中命中 v2，不影响正式入口绑定。
- 策略 C（禁止）：未完成回归或待验证项未通过时，禁止任何 v2 命中。

能力开关粒度（必须独立）：

- `editor.c06`
- `hunter.c11`
- `editor.c05`
- `editor.c13`

说明：禁止使用“一个总开关覆盖所有能力”的方式，避免回滚粒度过粗。

开启前置条件（必须全满足）：

- 关键项 `S2/H5/E3/P4/X4` 回归全通过。
- `C11/C05/C13` 完成最小样例验证并结论可复核。
- 已准备回滚证据路径（日志、配置快照、操作记录）。

最小上线顺序（必须按序）：

1. `shadow/test-binding` 验证
2. 显式 opt-in（`--agent *_v2`）
3. 保持默认入口不变，不进入默认自动命中

## 3) Kill-Switch 入口（三级）

### K1：路由回切（首选，分钟级）

- 目的：确保外部流量立即回到稳定入口。
- 操作（示例）：
  - `docker exec agent_argus openclaw agents bind --agent steward --bind telegram:default`
  - `docker exec agent_argus openclaw agents list --bindings --json`
  - `docker exec agent_argus openclaw channels status --json --probe`
- 成功标准：
  - `telegram:default` 明确绑定回 `steward`
  - 通道探活 `probe.ok=true`

### K2：能力冻结（配置级）

- 目的：立即停止 v2 能力命中（包括显式测试入口）。
- 操作原则：
  - 停止发放 v2 评审白名单。
  - 按能力开关粒度关闭（`editor.c06` / `hunter.c11` / `editor.c05` / `editor.c13`）。
  - 将本轮待验证能力标记为“冻结”，恢复到 v1 能力集。
- 成功标准：
  - 新请求无法命中 v2 能力。
  - 回归执行记录中可见冻结动作与时间戳。

### K3：运行态回退（兜底）

- 目的：当 K1/K2 无法稳定时，回退运行态到上一个稳定快照。
- 操作原则：
  - 使用既有恢复方案与回滚流程执行。
  - 回退后优先做 `steward` 冒烟与通道探活。
- 成功标准：
  - 关键指标恢复到基线范围。
  - 关键项一票否决问题消失。

## 4) 回滚触发阈值（沿用工程观察席建议）

- 安全：出现一次高危越权/敏感泄露即回滚。
- 稳定性：5xx 或不可用错误率 >= 0.5%（连续 5 分钟）即回滚。
- 性能：p95 延迟较基线上升 >= 30%（连续 10 分钟）即回滚。
- 质量：核心质量指标下降超过约定阈值（例如 Top-1 下降 >= 2pp）即回滚。
- 成本：单位请求成本较基线上升 >= 50%（连续 30 分钟且无收益证明）即回滚。

## 5) 执行留痕要求

- 每次启停 v2 都必须记录：
  - 操作人、时间、触发原因
  - 操作命令/配置变更
  - 回归结果与是否触发否决项
- 每次命中建议最小观测字段：
  - `request_id`
  - `role_id`
  - `intent_id`
  - `spec_version`
  - `experiment_bucket`
  - `latency_ms`
  - `cost`
  - `fail_reason`
- 记录建议统一落到：
  - `design/validation/gate3-regression-run-record-template-v1.md`

- v2 自动命中：关闭
