## 1) 工程判定

- 结论：**Conditional-Go**
- 说明：允许进入正式 `plan-eng-review`，但需先满足工程前置门槛

## 2) 进入正式评审前的 5 个工程门槛

1. `RoleSpec` 可校验化：模板需落到可机器校验 schema（含版本、字段约束）  
2. 意图与角色稳定 ID：候选意图与映射需具备稳定编号、弃用与兼容策略  
3. 黄金回归测试化：从清单转为可执行测试集（含期望、容差、失败 diff）  
4. 可控入口与熔断：默认入口不变，v2 需具备可控命中与 kill-switch  
5. 最小观测集：补齐 request_id/role_id/intent_id/耗时/失败原因/成本估算等埋点

## 3) C11 / C05 / C13 最小验证建议

- C11（深度调查与证据链）：
  - 先跑意图路由准确性验证（建议 Top-1 >= 95% 或 Top-3 >= 99%）
- C05（重复度/同质化风险提示）：
  - 用高风险提示样本集验证“0 越权输出 + 风险提示格式正确”
- C13（人性化改写）：
  - 跑新旧链路对照回归（建议回归通过率 >= 98%）

## 4) 回滚硬阈值（建议）

- 安全一票否决：出现一次高危越权/敏感泄露即回滚  
- 稳定性：5xx 或不可用错误率 >= 0.5%（连续 5 分钟）即回滚  
- 性能：p95 延迟较基线上升 >= 30%（连续 10 分钟）即回滚  
- 质量：核心质量指标下降超过约定阈值（例如 Top-1 下降 >= 2pp）即回滚  
- 成本：单位请求成本较基线上升 >= 50%（连续 30 分钟且无收益证明）即回滚
